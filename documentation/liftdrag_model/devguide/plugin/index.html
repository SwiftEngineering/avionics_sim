    Introduction

    This is a new lift-drag plugin for Gazebo. It implements the improved aerodynamic model whose mathematical basis is described here. The documentation for the C++ class that implements this new model documentation can be found here.

    The lift drag plugin is a subclass of Gazebo's ModelPlugin class.

    How To Use

    Add instance of plugin to link reference in SDF file

    Inside the element for a link you want to attach the lift drag plugin to, add a plugin element with filename of libLiftDrag_Enhanced_Plugin.so. This element should have the following subelements: Aero_LUT_alpha (lookup table values for angle of attack), Aero_LUT_CL (lookup table values for coefficients of lift), Aero_LUT_CD (lookup table values for coefficients of drag), link_name (name of associated link), control_joint_name (name of control joint, if applicable), area (surface area), cp (coordinates for center of pressure), forward (forward unit vector), upward (upward unit vector).

    Initialize LiftDrag model class in Load

    Initialize the class with values that should remain static during the simulation.

    Values that should remain static are element area, element lateral area, lookup tables, forward vector, and upward vector. Once these values are set, calculate the port vector to complete initialization.

    void LiftDrag_Enhanced_Plugin::Load(physics::ModelPtr _model, sdf::ElementPtr _sdf)
{

  //Set area and, if applicable, lateral area.
  try
  {
    if (_sdf->HasElement("area"))
    {
      this->area = _sdf->Get<double>("area");
      LiftDragModel.setArea(this->area);
    }

    // If lateral area specified, set the value.
    if (_sdf->HasElement("lateral_area"))
    {
      this->lateral_area = _sdf->Get<double>("lateral_area");
      LiftDragModel.setLateralArea(lateral_area);
    }
  }
  catch(const Lift_drag_model_exception& e)
  {
    if (e.isCritical())
    {
      gzerr<<e.what()<<std::endl;
    }
    else
    {
      gzwarn<<e.what()<<std::endl;
    }
  }


  // Load aerodynamic coefficient look up table elements from sdf
  std::string sdfString = ""; 


  if (_sdf->HasElement("Aero_LUT_alpha"))
  {
      // load parameters into string
      getSdfParam<std::string>(_sdf, "Aero_LUT_alpha", sdfString, sdfString);
      // convert from string into vector
     avionics_sim::Bilinear_interp::get1DLUTelementsFromString(sdfString, &LUTalpha);
  }

  if (_sdf->HasElement("Aero_LUT_CL"))
  {
    getSdfParam<std::string>(_sdf, "Aero_LUT_CL", sdfString, sdfString);
    avionics_sim::Bilinear_interp::get1DLUTelementsFromString(sdfString, &LUTCL);
  }

  if (_sdf->HasElement("Aero_LUT_CD"))
  {
    getSdfParam<std::string>(_sdf, "Aero_LUT_CD", sdfString, sdfString);
    avionics_sim::Bilinear_interp::get1DLUTelementsFromString(sdfString, &LUTCD);
  }

  // Set LUTs for lift drag model.
  LiftDragModel.setLUTs(LUTalpha, LUTCL, LUTCD);

  // Set upward and forward vectors
  if (_sdf->HasElement("forward"))
     LiftDragModel.setForwardVector(_sdf->Get<ignition::math::Vector3d>("forward"));

  if (_sdf->HasElement("upward"))
     LiftDragModel.setUpwardVector(_sdf->Get<ignition::math::Vector3d>("upward"));

  //Calculate port vector
  LiftDragModel.calculatePortVector();
}

    Update values during simulation time step

    In OnUpdate, update the world velocity and pose, then calculate wind angles and velocities. If applicable, update velocities and angles according to cases of propeller wash and control surface. Then, calculate the resultant force by calling calculateLiftDragModelValues. At the end of the time step, apply the resultant force to the center of pressure.

    World velocity can be updated by calling setWorldVelocity and passing in a Vector3d object representing world velocity. World pose is updated by calling setWorldPose.

    void LiftDrag_Enhanced_Plugin::OnUpdate(const common::UpdateInfo& _info)
{
  double lift, drag, lateral_force;
  bool calculateRotatedForces=true;
  ignition::math::Vector3d force;

  // Get world linear velocity
  ignition::math::Vector3d worldForces = this->link_->WorldForce();
  ignition::math::Vector3d vel = this->link_->WorldLinearVel(this->cp);
  LiftDragModel.setWorldVelocity(vel);

  // Get pose of body
  ignition::math::Pose3d pose = this->link_->WorldPose();

  //Set pose and velocity within model.
  LiftDragModel.setWorldPose(pose);

  try
  {
    // Calcuate Angles and Velocities
    LiftDragModel.calculateWindAnglesAndLocalVelocities();

    // If there is prop wash (motor output at this joint), examine motor exit velocity.
    if(motorOutSub != NULL)
    {
      /* If motor exit velocity is greater than the calculated one, 
      use motor exit velocity instead of freestream velocity.*/
      if (this->motorExitVelocity > LiftDragModel.getFreeStreamVelocity())
      {
        LiftDragModel.setFreeStreamVelocity(this->motorExitVelocity);
      }

      //Set the alpha under propwash to zero.
      LiftDragModel.setAlpha(0,true);
    }

    // If this is a control surface, set alpha from control surface deflection angle in Gazebo
    if(this->controlJoint)
    {
      double controlAngle = 0.0 ;
      controlAngle = -this->controlJoint->Position();
      LiftDragModel.setAlpha(controlAngle, true);
    }

    /* Set air density. If rho will remain constant during the simulation, 
    set air density in the Load method. 
    Future releases will include dynamic air density, hence initialization here. */
    rho = 1.225;
    LiftDragModel.setAirDensity(rho);

    // Calculate Lift and Drag according to aerodynamic element type
    if (this->controlJoint)
    {
      calculateRotatedForces=false;
    }
    else
    {
      calculateRotatedForces=true;
    }

    //Calculate coefficients and forces
    LiftDragModel.calculateLiftDragModelValues(calculateRotatedForces);
  }

  //Handle any caught exceptions
  catch(const Lift_drag_model_exception& e)
  {
    if (e.isCritical())
    {
      //
    }
    else
    {
      //
    }
  }

  // Construct force vector for application to link
  force = LiftDragModel.getForceVector();

  // Ensure numbers at cp are not NaN
  this->cp.Correct();

  // Add force in body frame, at position of CP.
  link_->AddLinkForce(force, this->cp);
}

    Calculation Steps

    The lift drag plugin performs the first and last of the ten steps outlined in the lift drag mathematical model.

    Receive vehicle pose and world linear velocity from Gazebo

    This is done by obtaining the world pose and world linear velocity from the link in OnUpdate and passing the values to the instance of the lift drag model.

    void LiftDrag_Enhanced_Plugin::OnUpdate(const common::UpdateInfo& _info)
{
  double lift, drag, lateral_force;
  bool calculateRotatedForces=true;
  ignition::math::Vector3d force;

  // Get world linear velocity
  ignition::math::Vector3d worldForces = this->link_->WorldForce();
  ignition::math::Vector3d vel = this->link_->WorldLinearVel(this->cp);
  LiftDragModel.setWorldVelocity(vel);

  // Get pose of body
  ignition::math::Pose3d pose = this->link_->WorldPose();

  //Set pose and velocity within model.
  LiftDragModel.setWorldPose(pose);

  //...
 }

    Apply the aerodynamic force, in body frame, to the vehicle

    This is done by calling the LiftDragModel class method getForceVector to obtain the resultant force, then applying the force to the center of pressure by calling the link instance method AddLinkForce.

    void LiftDrag_Enhanced_Plugin::OnUpdate(const common::UpdateInfo& _info)
{
  //...

  // Construct force vector for application to link
  force = LiftDragModel.getForceVector();

  // Ensure numbers are not NaN
  this->cp.Correct();

  // Add force in body frame, at position of CP.
  link_->AddLinkForce(force, this->cp);
 }
