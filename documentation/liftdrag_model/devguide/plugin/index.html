    Introduction

    This is a new lift-drag plugin for Gazebo. It implements the improved aerodynamic model whose mathematical basis is described here. The documentation for the C++ class that implements this new model documentation can be found here.

    The lift drag plugin is a subclass of Gazebo's ModelPlugin class.

    How To Use

    Add instance of plugin to link reference in SDF file

    Inside the element for a link you want to attach the lift drag plugin to, add a plugin element with filename of libLiftDrag_Enhanced_Plugin.so. This element should have the following subelements: Aero_LUT_alpha (lookup table values for angle of attack), Aero_LUT_CL (lookup table values for coefficients of lift), Aero_LUT_CD (lookup table values for coefficients of drag), link_name (name of associated link), control_joint_name (name of control joint, if applicable), area (surface area), cp (coordinates for center of pressure), forward (forward unit vector), upward (upward unit vector).

    Initialize LiftDrag model class in Load

    Initialize the class with values that should remain static during the simulation.

    Values that should remain static are element area, element lateral area, lookup tables, forward vector, and upward vector. Once these values are set, calculate the port vector to complete initialization.

    void LiftDrag_Enhanced_Plugin::Load(physics::ModelPtr _model, sdf::ElementPtr _sdf)
{
  GZ_ASSERT(_model, "LiftDragEnhancedPlugin _model pointer is NULL");
  GZ_ASSERT(_sdf, "LiftDragEnhancedPlugin _sdf pointer is NULL");
  GZ_ASSERT(_model->GetWorld(), "LiftDrag_Enhanced_Plugin world pointer is NULL");
  GZ_ASSERT(_model->GetWorld()->Physics(),
            "LiftDrag_Enhanced_Plugin physics pointer is NULL");

  model_ = _model;
  sdf = _sdf;
  world = model_->GetWorld();
  physics = world->Physics();

  SetNamespaceFromModel("robotNamespace");

  SetControlJointFromModel("control_joint_name");

  SetLinkFromModel("link_name");

  SetAerodynamicCenterFromModel("cp");

  SetPropWashFromModel("prop_wash", "motor");

  InitializeAerodynamicModel("area", "lateral_area", "Aero_LUT_alpha", "Aero_LUT_CL", 
"Aero_LUT_CD", "forward", "upward");

  InitializePublisher();

  InitializeUpdateConnection();
}

    Update values during simulation time step

    In OnUpdate, update air density, world velocity and pose, then call the appropriate form of the method updateForcesInBody_N depending on the presence of a motor and whether or not the element is a control surface. At the end of the time step, call getForceVector to get the resultant force, then apply it to the center of pressure.

    World velocity can be updated by calling setWorldVelocity and passing in a Vector3d object representing world velocity. World pose is updated by calling setWorldPose.

    void LiftDrag_Enhanced_Plugin::OnUpdate() {
  // Get world linear valocity and pose
  ignition::math::Vector3d worldForces = link_->WorldForce();
  ignition::math::Vector3d velocityInWorld_m_per_s = link_->WorldLinearVel(
        aerodynamicCenter_m);
  ignition::math::Pose3d poseInWorld_m_rad = link_->WorldPose();

  ignition::math::Vector3d aeroForceOnBody_N;
  if (motorOutSub != NULL && controlJoint) {
    double propWash_m_per_s = this->motorExitVelocity_m_per_s;
    double controlAngle_rad = -controlJoint->Position();
    aeroForceOnBody_N = aerodynamicModel.updateForcesInBody_N(propWash_m_per_s, 
propWash_m_per_s, 
RAD2DEG(controlAngle_rad), 
 0.0);
  } else {
    aeroForceOnBody_N = aerodynamicModel.updateForcesInBody_N(poseInWorld_m_rad,
    velocityInWorld_m_per_s);
  }

  // Ensure that cp is not NaN
  this->aerodynamicCenter_m.Correct();

  // Add force in frame of link, at position of CP.
  link_->AddLinkForce(aeroForceOnBody_N, this->aerodynamicCenter_m);

  // TODO: Add moment since we are applying force at aerodynamic center.

  RecordState();
}

    The lift drag plugin performs the first and last of the ten steps outlined in the lift drag mathematical model.

    Apply the aerodynamic force, in body frame, to the vehicle

    This is done by passing the returned resultant force from updateForcesInBody_N and the aerodynamic center of the link to method ApplyForce.

    void LiftDrag_Enhanced_Plugin::OnUpdate() {
  //...

  ignition::math::Vector3d aeroForceOnBody_N;
  if (motorOutSub != NULL && controlJoint) {
    double propWash_m_per_s = this->motorExitVelocity_m_per_s;
    double controlAngle_rad = -controlJoint->Position();
    aeroForceOnBody_N = aerodynamicModel.updateForcesInBody_N(propWash_m_per_s, 
propWash_m_per_s, 
RAD2DEG(controlAngle_rad), 
 0.0);
  } else {
    aeroForceOnBody_N = aerodynamicModel.updateForcesInBody_N(poseInWorld_m_rad,
    velocityInWorld_m_per_s);
  }

  // Ensure that cp is not NaN
  this->aerodynamicCenter_m.Correct();

  // Add force in frame of link, at position of CP.
  link_->AddLinkForce(aeroForceOnBody_N, this->aerodynamicCenter_m);

  // TODO: Add moment since we are applying force at aerodynamic center.

  RecordState();
}
